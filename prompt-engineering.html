How it works

Four moves that turn a chat into a factory

The difference between "asked an LLM for a website" and a repeatable system is structure. These are the layers I engineer.

01 / Operating manual

Persona & rules

A dense system prompt that fixes role, output density, hard bans and a verify-before-ship pipeline — so quality is the default, not luck.

02 / Design DNA

Reusable vocabulary

A 200-token style vocabulary (layout, motion, palette, typographic feel) the model composes from — making outputs varied yet on-brand.

03 / Structured output

Schema & manifest

Artifacts emit to a fixed schema (JSON manifest) so a static site can index, filter and live-preview them with zero manual wiring.

04 / Agent harness

Build & verify loop

Multi-step agents scaffold, run and self-check the result — catching format drift and broken previews before anything ships.

The prompts

Copy-paste the actual system

Real, reusable prompts behind the work above — the operating layer, the design-DNA generator, and the inline-preview gallery spec. Hit copy and run them.

01 · Operating manual (system prompt)Meta-prompt
You are a senior design+front-end engineer. Operate at MAX reasoning, MIN tokens.

RULES
- Lead with the artifact, not preamble. No "Great question", no recap.
- One self-contained file unless told otherwise. No `...`, no stubs — full code.
- Distinctive over generic: avoid default-AI gradients, centered-everything, lorem.
- Numbers over adjectives. Tables over paragraphs for >=3 options.

PIPELINE (every output): Understand -> Structure -> Build -> Self-verify.
Before shipping, check: does it run, is it responsive, does the preview render?
If you cannot defend a choice, change it. Ship the version you'd put your name on.
02 · Design-DNA generatorFew-shot · style transfer
Generate ONE landing page from this design DNA. Compose, don't copy.

DNA:
- layout      : {bento | editorial | terminal | swiss-grid}
- motion      : {scroll-reveal, magnetic buttons, cursor spotlight}
- palette     : warm-canvas #EFEBE2 / ink #13110D / accent #1B3FE0 / clay #D8623A
- type        : Fraunces (display) + Space Grotesk (UI) + JetBrains Mono (meta)
- feel        : "editorial, confident, expensive — not a template"

OUTPUT: a single self-contained .html (inline CSS+JS). Mobile-first, a11y-clean,
no external build. End with a one-line note: which DNA traits you leaned on.
03 · Inline-preview gallery specStructured output
Build a static gallery that indexes many HTML artifacts.

REQUIREMENTS
- Read artifacts from a JSON manifest: [{ id, title, category, type, path }].
- Each card shows a LIVE inline iframe preview (scaled), never a click-to-open modal.
- Mount the iframe only when the card nears the viewport via IntersectionObserver,
  and UNMOUNT it when it scrolls away — keep only ~10-20 heavy iframes alive at once.
- Filter chips by category + a "/" search box. Zero dependencies, single index.html.
- A separate node script scans the repo and writes manifest.json (re-runnable).

Deliver: index.html + build-manifest.js, both complete.
